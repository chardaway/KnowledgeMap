# 线性 — 垂直因果链深挖

逐层追问「为什么」，构建因果链知识图谱。

## 触发条件

当用户表达以下意图时触发：
- 深层分析某个问题 / 现象的原因
- 追问根本原因、追根溯源
- 使用「线性」「因果链」「深挖」「为什么链」等关键词

## 核心规则

1. **一文件一节点**：每个原因层级生成一个独立的 md 文件
2. **链式链接**：父节点 → 子节点（`[[子节点名]]`），子节点 → 父节点（`[[父节点名]]`）
3. **逐步推进**：每生成一个节点后停下来，等待用户决定是否继续
4. **不预设深度**：用户决定何时停止
5. **不修改已有笔记**：仅创建新文件，不修改 vault 中任何已有文件

## 工作流程

### 步骤 1：读取配置

读取项目根目录 `D:\KnowledgeMap\config.json`，获取：
- `vault_path`：Obsidian vault 根目录
- `output_folder`：输出文件夹名（默认 `认知地图`）

输出目录为：`{vault_path}/{output_folder}/线性/`

### 步骤 2：接收问题

用户提出一个待分析的问题。以此问题作为根节点标题。

### 步骤 3：创建根节点

在输出目录下创建 `{问题标题}.md`：

```markdown
---
tags:
  - 认知地图
  - 线性
created: {当前时间}
modified: {当前时间}
---
# {问题标题}

{对问题的简要阐述与分析}
```

文件名规则：
- 使用问题标题直接作为文件名
- 去除 Windows 非法字符 `\ / : * ? " < > |`
- 对截断过长文件名和冲突做处理

### 步骤 4：呈现并等待

向用户呈现分析结果，然后询问：「**是否继续深挖原因？**」

### 步骤 5：深挖一层（用户确认后）

1. 分析当前节点的深层原因，提炼出一个简洁的原因陈述作为子节点标题
2. 在输出目录下创建 `{原因标题}.md`
3. **更新父节点**：在父节点文件底部「---」分隔线后的链接区添加 `[[子节点名]]`
4. 子节点底部链接区写 `[[父节点名]]`

子节点格式：

```markdown
---
tags:
  - 认知地图
  - 线性
created: {当前时间}
modified: {当前时间}
---
# {原因标题}

{对该层原因的详细分析}

---

[[父节点名]]
```

### 步骤 6：循环

重复步骤 4-5，每次深挖一层，形成链式结构：

```
问题A  ←→  原因B  ←→  原因C  ←→  ...
```

每个节点的底部链接区结构不同：
- **中间节点**（如原因B）：`[[上一个节点]] · [[下一个节点]]`
- **根节点**：`[[下一个节点]]`（只有向下）
- **叶子节点**：`[[上一个节点]]`（只有向上）

### 停止条件

- 用户说「停」「够了」「不用了」
- 用户提出新的话题（切换到其他技能）

## 链接规范

- 只用简单 `[[文件名]]` 形式，不加别名、标题锚点、块引用
- 链接放在文件底部 `---` 分隔线后，用 ` · ` 分隔多个链接
- 父节点中追加子链接时，用 Write 工具读取当前内容后在链接区追加

## 示例对话

```
用户: 线性：为什么我的笔记系统总是用不起来？

Claude: [创建 为什么笔记系统用不起来.md]
       [分析原因：缺乏明确的输入→输出流程...]
       是否继续深挖原因？

用户: 继续

Claude: [创建 缺乏明确的知识处理流程.md]
       [更新父节点链接]
       [分析：因为没有定义笔记的 lifecycle...]
       是否继续深挖原因？

用户: 停
```
