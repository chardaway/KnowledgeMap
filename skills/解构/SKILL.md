# 解构 — 星形逻辑拆解

将一篇已有笔记拆解为「核心问题 + 相关问题」的星形结构。

## 触发条件

当用户表达以下意图时触发：
- 将一篇笔记拆分成多个相关笔记
- 对某个主题做结构化分解
- 使用「解构」「拆解」「分解」「结构化」等关键词

## 核心规则

1. **星形拓扑**：一个核心节点，周围 3-7 个相关子节点
2. **所有链接指向核心**：`相关 ← 核心 → 相关`
3. **子节点之间不互链**
4. **不修改用户原始笔记**：只读取分析，不写入
5. **生成前需确认**：向用户展示核心问题和子问题列表，经确认后再创建文件

## 工作流程

### 步骤 1：读取配置

读取 `D:\KnowledgeMap\config.json`，获取 `vault_path` 和 `output_folder`。

### 步骤 2：定位源笔记

用户提供一个 vault 中的笔记名称。按以下优先级查找：

1. `obsidian search query="笔记名" vault="chardaway"`（CLI 搜索）
2. 或直接 `vault_path/**/笔记名.md` 搜索

找到后读取笔记全部内容。

### 步骤 3：分析并建议结构

根据笔记内容：
1. 提炼一个**核心问题**（笔记试图回答的根本问题）
2. 生成 3-7 个**相关子问题**（核心问题的不同侧面、前提、推论或应用）

向用户展示：

```
核心问题：xxx
相关问题：
  1. xxx
  2. xxx
  3. xxx
  ...

确认创建？(可修改核心问题或子问题后确认)
```

### 步骤 4：创建文件（用户确认后）

输出目录：`{vault_path}/{output_folder}/解构/`

#### 4.1 创建核心节点

文件名：`{output_folder}/解构/{核心问题}.md`

```markdown
---
tags:
  - 认知地图
  - 解构
created: {当前时间}
modified: {当前时间}
---
# {核心问题}

{对核心问题的阐述}

> 源自：[[{源笔记名}]]

---

[[子问题1]] · [[子问题2]] · [[子问题3]] · ...
```

#### 4.2 创建每个子节点

文件名：`{output_folder}/解构/{子问题}.md`

```markdown
---
tags:
  - 认知地图
  - 解构
created: {当前时间}
modified: {当前时间}
---
# {子问题}

{对该子问题的简要分析}

> 源自：[[{核心问题}]]

---

[[{核心问题}]]
```

### 搭建关系总结

```
         ┌→ [[子问题1]]  ─ 回链 → [[核心]]
         │
[[核心]] ─┼→ [[子问题2]]  ─ 回链 → [[核心]]
         │
         └→ [[子问题3]]  ─ 回链 → [[核心]]
```

## 链接规范

- 仅使用简单 `[[文件名]]` 形式
- 核心节点底部列出所有子节点链接
- 每个子节点底部仅链接回核心节点
- 子节点之间不互相链接

## 文件命名

- 使用中文标题直接作为文件名
- 去除 Windows 非法字符 `\ / : * ? " < > |`
- 如有重名冲突，追加 `-2`、`-3` 后缀

## 示例对话

```
用户: 解构「忠诚的悖论」

Claude: [读取 忠诚的悖论.md]
        分析完成，建议结构如下：

        核心问题：忠诚是一种美德还是盲从？
        相关问题：
          1. 忠诚的对象是否有道德边界？
          2. 背叛是否有时是更道德的选择？
          3. 忠诚与独立思考是否必然冲突？
          4. 制度忠诚与个人忠诚有何不同？
          5. 历史上的忠诚观如何演变？

        确认创建？可以增删或修改任何条目。

用户: 把第5个改成「忠诚观的文化差异」，确认。

Claude: [创建 6 个 md 文件，建立星形链接]
        已创建：
        - 忠诚是一种美德还是盲从？.md（核心）
        - 忠诚的对象是否有道德边界？.md
        - 背叛是否有时是更道德的选择？.md
        - 忠诚与独立思考是否必然冲突？.md
        - 制度忠诚与个人忠诚有何不同？.md
        - 忠诚观的文化差异.md
```
