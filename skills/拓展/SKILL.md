# 拓展 — 轮辐式横向发散

以任意笔记为核心，向外拓展一层相关问题。

## 触发条件

当用户表达以下意图时触发：
- 围绕一个话题发散思考
- 为某个笔记找关联问题
- 使用「拓展」「发散」「扩展」「关联」「脑暴」等关键词

## 核心规则

1. **轮辐拓扑**：所有生成文件回链到同一个核心节点
2. **只拓展一层**：每个生成文件是一个相关问题，不再继续深挖
3. **不修改核心笔记**：原始笔记保持原样
4. **生成 4-8 个相关问题**
5. **生成前需确认**：展示建议的问题列表，用户可增删修改

## 工作流程

### 步骤 1：读取配置

读取 `D:\KnowledgeMap\config.json`，获取 `vault_path` 和 `output_folder`。

### 步骤 2：定位核心笔记

用户指定 vault 中的一个笔记。查找方式：
1. `obsidian search query="笔记名" vault="chardaway"`
2. 或直接文件搜索

找到后读取内容，理解核心主题。

### 步骤 3：生成相关问题

基于核心笔记的内容，横向思考生成 4-8 个相关问题。方向可包括：

- **类比**：类似现象在其他领域如何表现？
- **推论**：如果这是真的，还意味着什么？
- **反例**：什么情况下这不成立？
- **应用**：这个想法可以怎么用？
- **前提**：这个观点建立在什么假设上？
- **对比**：与之相反的观点是什么？

展示给用户确认：

```
核心：「{核心笔记名}」

建议的相关问题：
  1. xxx
  2. xxx
  ...

确认创建？(可增删修改)
```

### 步骤 4：创建文件（用户确认后）

输出目录：`{vault_path}/{output_folder}/拓展/`

每个相关问题的文件格式：

```markdown
---
tags:
  - 认知地图
  - 拓展
created: {当前时间}
modified: {当前时间}
---
# {问题标题}

> 源自：[[{核心笔记名}]]

{对该问题的简要分析}

---

[[{核心笔记名}]]
```

### 拓扑总结

```
              ┌── [[问题1]] ──┐
              ├── [[问题2]] ──┤
[[核心笔记]]  ←── [[问题3]] ──→  [[核心笔记]]
              ├── [[问题4]] ──┤
              └── [[问题5]] ──┘
```

每个新文件仅链接回核心，彼此之间不互链。

## 链接规范

- 仅使用简单 `[[文件名]]` 形式
- 每个拓展文件只包含一个链接：回链到核心笔记
- 不修改核心笔记文件

## 示例对话

```
用户: 拓展「文明6时代之熵」

Claude: [读取 文明6时代之熵.md]
        围绕「文明6时代之熵」，建议以下拓展方向：

        1. 文明中的熵增可逆吗？
        2. 其他策略游戏是否有类似的时代机制？
        3. 现实文明中的兴衰也遵循熵模式吗？
        4. 从熵的角度看，科技树是加速还是延缓崩溃？
        5. 有哪些文明成功逆转了衰退？
        6. 个体玩家行为和文明级熵增有什么关系？

        确认创建？可修改列表。

用户: 去掉第6个，确认。

Claude: [创建 5 个 md 文件]
        已创建：
        - 文明中的熵增可逆吗？.md
        - 其他策略游戏是否有类似的时代机制？.md
        - 现实文明中的兴衰也遵循熵模式吗？.md
        - 从熵的角度看，科技树是加速还是延缓崩溃？.md
        - 有哪些文明成功逆转了衰退？.md

        所有文件均链接回 [[文明6时代之熵]]
```
