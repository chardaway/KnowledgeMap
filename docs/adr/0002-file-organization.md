# ADR-0002: 生成文件的组织方式

## 状态
已接受

## 日期
2026-05-20

## 背景
三个技能会在 Obsidian vault 中创建新的 md 文件。这些文件需要和用户原有笔记区分开，便于管理。

## 决策

在 vault 根目录下创建专门的 `认知地图/` 文件夹，按技能分为三个子目录：

```
认知地图/
  线性/    # 因果链节点
  解构/    # 星形拆解节点
  拓展/    # 轮辐发散节点
```

文件直接以中文标题命名（如 `为什么天空是蓝色的？.md`）。

## 理由
- 与用户年度目录（2025/、2026/）完全分离，不会污染原有笔记
- Obsidian 图谱视图会自动显示跨文件夹的链接关系
- 按技能分类，方便查找和清理
- 中文标题与用户现有命名习惯一致

## 备选方案
- **混入年份目录**: 会与用户笔记混杂，难以区分和管理
- **前缀命名**（如 `lin-xxx.md`）: 冗余，子目录已提供分类信息

## 影响
- 所有技能的输出路径规则统一
- `config.json` 中 `output_folder` 控制根目录名
- 不修改用户原始笔记，仅创建新文件
